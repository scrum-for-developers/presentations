Scrum for developers - Sprint 1

Git, Maven & Jenkins

Version Control

Advantages:
  • Share my code with my peers
  • See what others contributed
  • Get new changes from others
  • Browse the codebase
  • Go back in time (virtually)
  • Experiment with out affecting others
 Definition of Done:
  • sources are under version control

We will look more into version control in sprint 2 - especially feature branches vs. trunk based development

Build & Dependency Management

Advantages:
  • Quickly find and add new dependencies
  • Find out transitive dependencies
  • Detect dependency conflicts
  • All projects follow the same conventions
  • Allows self-contained CI
 Definition of Done:
  • dependencies are managed via maven
  • sources are under version control

Continuous Integration

Advantages:
  • Fast feedback for everybody about integration status
  • Picks up changes to version control automatically
  • Uses same build environment as local developer
  • Single point of truth (no more "works on my machine")
 Definition of Done:
  • project is built on CI-server
  • dependencies are managed
  • sources are under version control
